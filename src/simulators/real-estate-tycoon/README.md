# 房地产大亨模拟器代码地图

这个模拟器按“经营桌面”拆分。以后新增机制先放进对应 `systems/` 子目录，不继续把大块逻辑直接塞进 `app.js`。

## 顶层文件

- `index.html`：页面骨架和脚本入口。
- `style.css`：当前仍是主样式文件；新增大块 UI 后续按系统拆 CSS。
- `data.js`：剧情事件、角色台词、初始数据等内容库。
- `app.js`：运行入口、状态机、渲染调度和旧代码兼容层。它应该越来越薄，只负责把系统模块串起来。

## 系统目录

- `systems/auction/`：土拍、地块、竞拍轮次、联合竞标、成交归属。
- `systems/finance/`：银行、朋友、小贷、高息短钱、债务和付息。
- `systems/relation/`：朋友、政府、银行、其他关系；借钱报价、共同开发比例谈判。
- `systems/project/`：项目开发、宣发、监工、监管户、总包节点款、交付。

每个系统目录都有自己的 `README.md`。新增文件时先更新对应 README，说明这个文件负责什么，避免后面继续变成一个巨型文件。

## 突发事件因果规则

- 不把突发事件当纯随机题库。玩家做过的选择必须留下前因，并影响后续事件权重。
- 事件队列需要保存来源：哪一回合、哪个事件、哪个选项、哪个人物/项目、为什么会发酵。
- 事件展示时要让玩家看懂因果：不是“突然来了”，而是“你之前做了什么，所以现在谁来找你”。
- 只有宏观周期、政策收紧、银行行业性抽贷这类外部冲击可以相对随机；即便如此，也要受当前现金、债务、项目、关系和暗账状态加权。
